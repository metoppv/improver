27 """Module to apply thresholding to a parameter dataset. 

28 

29 Calculate the threshold truth values of input data relative to the 

30 provided threshold value. A fuzzy factor or fuzzy bounds may be provided 

31 to smooth probabilities where values are close to the threshold. 

32 

33 Args: 

34 cube (iris.cube.Cube): 

35 A cube to be processed. 

36 land_sea_mask (Cube): 

37 Binary land-sea mask data. True for land-points, False for sea. 

38 Restricts in-vicinity processing to only include points of a 

39 like mask value. 

40 threshold_values (list of float): 

41 Threshold value or values (e.g. 270K, 300K) to use when calculating 

42 the probability of the input relative to the threshold value(s). 

43 These are provided as a comma separated list, e.g. 270,300 

44 The units of these values, e.g. K in the example can be defined 

45 using the threshold_units argument or are otherwise assumed to 

46 match the units of the diagnostic being thresholded. 

47 threshold_values and and threshold_config are mutually exclusive 

48 arguments, defining both will lead to an exception. 

49 threshold_config (dict): 

50 Threshold configuration containing threshold values and 

51 (optionally) fuzzy bounds. Best used in combination with 

52 'threshold_units'. It should contain a dictionary of strings that 

53 can be interpreted as floats with the structure: 

54 "THRESHOLD_VALUE": [LOWER_BOUND, UPPER_BOUND] 

55 e.g: {"280.0": [278.0, 282.0], "290.0": [288.0, 292.0]}, 

56 or with structure "THRESHOLD_VALUE": "None" (no fuzzy bounds). 

57 Repeated thresholds with different bounds are ignored; only the 

58 last duplicate will be used. 

59 threshold_values and and threshold_config are mutually exclusive 

60 arguments, defining both will lead to an exception. 

61 threshold_units (str): 

62 Units of the threshold values. If not provided the units are 

63 assumed to be the same as those of the input cube. Specifying 

64 the units here will allow a suitable conversion to match 

65 the input units if possible. 

66 comparison_operator (str): 

67 Indicates the comparison_operator to use with the threshold. 

68 e.g. 'ge' or '>=' to evaluate data >= threshold or '<' to 

69 evaluate data < threshold. When using fuzzy thresholds, there is 

70 no difference between < and <= or > and >=. 

71 Options: > >= < <= gt ge lt le. 

72 fuzzy_factor (float of None): 

73 A decimal fraction defining the factor about the threshold value(s) 

74 which should be treated as fuzzy. Data which fail a test against 

75 the hard threshold value may return a fractional truth value if 

76 they fall within this fuzzy factor region. 

77 Fuzzy factor must be in the range 0-1, with higher values 

78 indicating a narrower fuzzy factor region / sharper threshold. 

79 A fuzzy factor cannot be used with a zero threshold or a 

80 threshold_config file. 

81 collapse_coord (list of str): 

82 A coordinate or list of coordinates over which an average is 

83 calculated by collapsing these coordinates. The only supported 

84 options are combinations of "realization" and "percentile" with 

85 "time"; realization and percentile cannot both be collapsed. 

86 If "percentile" is included the percentile coordinate will be 

87 rebadged as a realization coordinate prior to collapse. 

88 The percentile coordinate needs to be evenly spaced around the 

89 50th percentile to allow successful conversion from percentiles 

90 to realizations and subsequent collapsing over the realization 

91 coordinate. 

92 collapse_cell_methods (dict): 

93 An optional dictionary that describes cell methods to apply in 

94 relation to the collapsed coordinates. By default the threshold 

95 method does not return a cell method for collapsed coordinates. 

96 The dictionary should take the form: {"coord_name": "method"} 

97 with an entry for every coordinate for which a method is 

98 desired. 

99 vicinity (list of float / int): 

100 List of distances in metres used to define the vicinities within 

101 which to search for an occurrence. Each vicinity provided will 

102 lead to a different gridded field. 

103 fill_masked (float): 

104 If provided all masked points in cube will be replaced with the 

105 provided value before thresholding. 

106 

107 Returns: 

108 iris.cube.Cube: 

109 Cube of probabilities relative to the given thresholds 

110 """ 

111 from improver.threshold import Threshold 

112 

113 return Threshold( 

114 threshold_values=threshold_values, 

115 threshold_config=threshold_config, 

116 fuzzy_factor=fuzzy_factor, 

117 threshold_units=threshold_units, 

118 comparison_operator=comparison_operator, 

119 collapse_coord=collapse_coord, 

120 collapse_cell_methods=collapse_cell_methods, 

121 vicinity=vicinity, 

122 fill_masked=fill_masked, 

123 )(cube, land_sea_mask)
