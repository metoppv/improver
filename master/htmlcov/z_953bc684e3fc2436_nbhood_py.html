14def process(

17 *, 

18 neighbourhood_output, 

19 neighbourhood_shape="square",

21 lead_times: cli.comma_separated_list = None, 

22 degrees_as_complex=False, 

23 weighted_mode=False, 

24 area_sum=False,

28 """Runs neighbourhood processing. 

29 

30 Apply the requested neighbourhood method via the 

31 NeighbourhoodProcessing plugin to a Cube. 

32 

33 Args: 

34 cube (iris.cube.Cube): 

35 The Cube to be processed, usually a thresholded data set. 

36 mask (iris.cube.Cube): 

37 A cube to mask the input cube. The data should contain 1 for 

38 usable points and 0 for discarded points. 

39 Can't be used with "percentiles" as neighbourhood_output (Optional) 

40 neighbourhood_output (str): 

41 The form of the results generated using neighbourhood processing. 

42 If "probabilities" is selected, the mean probability with a 

43 neighbourhood is calculated. If "percentiles" is selected, then 

44 the percentiles are calculated with a neighbourhood. Calculating 

45 percentiles from a neighbourhood is only supported for a circular 

46 neighbourhood, and the input cube should be ensemble realizations. 

47 The calculation of percentiles from a neighbourhood is notably slower 

48 than neighbourhood processing using a thresholded probability field. 

49 Options: "probabilities", "percentiles". 

50 neighbourhood_shape (str): 

51 Name of the neighbourhood method to use. Only a "circular" 

52 neighbourhood shape is applicable for calculating "percentiles" 

53 output. 

54 Options: "circular", "square". 

55 Default: "square". 

56 radii (list of float): 

57 The radius or a list of radii in metres of the neighbourhood to 

58 apply. 

59 If it is a list, it must be the same length as lead_times, which 

60 defines at which lead time to use which nbhood radius. The radius 

61 will be interpolated for intermediate lead times. 

62 lead_times (list of int): 

63 The lead times in hours that correspond to the radii to be used. 

64 If lead_times are set, radii must be a list the same length as 

65 lead_times. 

66 degrees_as_complex (bool): 

67 Include this option to process angles as complex numbers. 

68 Not compatible with circular kernel or percentiles. 

69 weighted_mode (bool): 

70 Include this option to set the weighting to decrease with radius. 

71 Otherwise a constant weighting is assumed. 

72 weighted_mode is only applicable for calculating "probability" 

73 neighbourhood output using the circular kernel. 

74 area_sum (bool): 

75 Return sum rather than fraction over the neighbourhood area. 

76 percentiles (float): 

77 Calculates value at the specified percentiles from the 

78 neighbourhood surrounding each grid point. This argument has no 

79 effect if the output is probabilities. 

80 halo_radius (float): 

81 Set this radius in metres to define the excess halo to clip. Used 

82 where a larger grid was defined than the standard grid and we want 

83 to clip the grid back to the standard grid. Otherwise no clipping 

84 is applied. 

85 

86 Returns: 

87 iris.cube.Cube: 

88 A processed Cube. 

89 

90 Raises: 

91 RuntimeError: 

92 If weighted_mode is used with the wrong neighbourhood_output. 

93 RuntimeError: 

94 If degree_as_complex is used with 

95 neighbourhood_output='percentiles'. 

96 RuntimeError: 

97 If degree_as_complex is used with neighbourhood_shape='circular'. 

98 """ 

99 from improver.nbhood.nbhood import MetaNeighbourhood 

100 

101 plugin = MetaNeighbourhood( 

102 neighbourhood_output=neighbourhood_output, 

103 neighbourhood_shape=neighbourhood_shape, 

104 radii=radii, 

105 lead_times=lead_times, 

106 degrees_as_complex=degrees_as_complex, 

107 weighted_mode=weighted_mode, 

108 area_sum=area_sum, 

109 percentiles=percentiles, 

110 halo_radius=halo_radius, 

111 ) 

112 return plugin(cube, mask=mask)
