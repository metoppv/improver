18 times: cli.comma_separated_list = None, 

19 interpolation_method="linear",

24 """Interpolate data between validity times. 

25 

26 Interpolate data to intermediate times between the validity times of two 

27 cubes. This can be used to fill in missing data (e.g. for radar fields) 

28 or to ensure data is available at the required intervals when model data 

29 is not available at these times. 

30 

31 Args: 

32 start_cube (iris.cube.Cube): 

33 Cube containing the data at the beginning. 

34 end_cube (iris.cube.Cube): 

35 Cube containing the data at the end. 

36 interval_in_mins (int): 

37 Specifies the interval in minutes at which to interpolate between 

38 the two input cubes. 

39 A number of minutes which does not divide up the interval equally 

40 will raise an exception. 

41 If intervals_in_mins is set then times can not be used. 

42 times (str): 

43 Specifies the times in the format {YYYYMMDD}T{HHMM}Z 

44 at which to interpolate between the two input cubes. 

45 Where {YYYYMMDD} is year, month, day and {HHMM} is hour and minutes 

46 e.g 20180116T0100Z. More than one time can be provided separated 

47 by a comma. 

48 If times are set, interval_in_mins can not be used. 

49 interpolation_method (str): 

50 ["linear", "solar", "daynight"] 

51 Specifies the interpolation method; 

52 solar interpolates using the solar elevation, 

53 daynight uses linear interpolation but sets night time points to 

54 0.0 linear is linear interpolation. 

55 accumulation: 

56 Set True if the diagnostic being temporally interpolated is a 

57 period accumulation. The output will be renormalised to ensure 

58 that the total across the period constructed from the shorter 

59 intervals matches the total across the period from the coarser 

60 intervals. Trends between adjacent input periods will be used 

61 to provide variation across the interpolated periods. 

62 max: 

63 Set True if the diagnostic being temporally interpolated is a 

64 period maximum. Trends between adjacent input periods will be used 

65 to provide variation across the interpolated periods where these 

66 are consistent with the inputs. 

67 min: 

68 Set True if the diagnostic being temporally interpolated is a 

69 period minimum. Trends between adjacent input periods will be used 

70 to provide variation across the interpolated periods where these 

71 are consistent with the inputs. 

72 Returns: 

73 iris.cube.CubeList: 

74 A list of cubes interpolated to the desired times. The 

75 interpolated cubes will always be in chronological order of 

76 earliest to latest regardless of the order of the input. 

77 """ 

78 from improver.utilities.cube_manipulation import MergeCubes 

79 from improver.utilities.temporal import cycletime_to_datetime, iris_time_to_datetime 

80 from improver.utilities.temporal_interpolation import TemporalInterpolation 

81 

82 (time_start,) = iris_time_to_datetime(start_cube.coord("time")) 

83 (time_end,) = iris_time_to_datetime(end_cube.coord("time")) 

84 if time_end < time_start: 

85 # swap cubes 

86 start_cube, end_cube = end_cube, start_cube 

87 

88 if times is not None: 

89 times = [cycletime_to_datetime(timestr) for timestr in times] 

90 

91 result = TemporalInterpolation( 

92 interval_in_minutes=interval_in_mins, 

93 times=times, 

94 interpolation_method=interpolation_method, 

95 accumulation=accumulation, 

96 max=max, 

97 min=min, 

98 )(start_cube, end_cube) 

99 return MergeCubes()(result)
