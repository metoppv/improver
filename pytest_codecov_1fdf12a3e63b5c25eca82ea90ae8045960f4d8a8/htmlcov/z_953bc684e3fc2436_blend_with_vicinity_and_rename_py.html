14def process(

17 new_name: str = None, 

18 vicinity_radius: float = None, 

19 weighting_method="linear", 

20 weighting_coord="forecast_period", 

21 weighting_config: cli.inputjson = None, 

22 attributes_config: cli.inputjson = None,

24 y0val: float = None,

28 record_run_attr: str = None, 

29 spatial_weights_from_mask=False, 

30 fuzzy_length=20000.0, 

31): 

32 """Runs weighted blending, with additional options to handle mismatched in_vicinity coords 

33 and to apply a new name to the output cube. 

34 

35 Check for inconsistent arguments, then calculate a weighted blend 

36 of input cube data using the options specified. 

37 

38 Args: 

39 cubes (iris.cube.CubeList): 

40 Cubelist of cubes to be blended. 

41 coordinate (str): 

42 The coordinate over which the blending will be applied. 

43 new_name (str): 

44 Replaces cube.name() on the output cube. 

45 vicinity_radius (float): 

46 The data value associated with the vicinity coord on the output cube. 

47 Assumes same units as the x and y spatial coordinates. 

48 weighting_method (str): 

49 Method to use to calculate weights used in blending. 

50 "linear" (default): calculate linearly varying blending weights. 

51 "nonlinear": calculate blending weights that decrease 

52 exponentially with increasing blending coordinates. 

53 "dict": calculate weights using a dictionary passed in. 

54 weighting_coord (str): 

55 Name of coordinate over which linear weights should be scaled. 

56 This coordinate must be available in the weights dictionary. 

57 weighting_config (dict or None): 

58 Dictionary from which to calculate blending weights. Dictionary 

59 format is as specified in 

60 improver.blending.weights.ChoosingWeightsLinear 

61 attributes_config (dict): 

62 Dictionary describing required changes to attributes after blending 

63 cycletime (str): 

64 The forecast reference time to be used after blending has been 

65 applied, in the format YYYYMMDDTHHMMZ. If not provided, the 

66 blended file takes the latest available forecast reference time 

67 from the input datasets supplied. 

68 y0val (float): 

69 The relative value of the weighting start point (lowest value of 

70 blend coord) for choosing default linear weights. 

71 If used this must be a positive float or 0. 

72 ynval (float): 

73 The relative value of the weighting end point (highest value of 

74 blend coord) for choosing default linear weights. This must be a 

75 positive float or 0. 

76 Note that if blending over forecast reference time, ynval >= y0val 

77 would normally be expected (to give greater weight to the more 

78 recent forecast). 

79 cval (float): 

80 Factor used to determine how skewed the non-linear weights will be. 

81 A value of 1 implies equal weighting. 

82 model_id_attr (str): 

83 The name of the dataset attribute to be used to identify the source 

84 model when blending data from different models. 

85 record_run_attr: 

86 The name of the dataset attribute to be used to store model and 

87 cycle sources in metadata, e.g. when blending data from different 

88 models. Requires model_id_attr. 

89 spatial_weights_from_mask (bool): 

90 If True, this option will result in the generation of spatially 

91 varying weights based on the masks of the data we are blending. 

92 The one dimensional weights are first calculated using the chosen 

93 weights calculation method, but the weights will then be adjusted 

94 spatially based on where there is masked data in the data we are 

95 blending. The spatial weights are calculated using the 

96 SpatiallyVaryingWeightsFromMask plugin. 

97 fuzzy_length (float): 

98 When calculating spatially varying weights we can smooth the 

99 weights so that areas close to areas that are masked have lower 

100 weights than those further away. This fuzzy length controls the 

101 scale over which the weights are smoothed. The fuzzy length is in 

102 terms of m, the default is 20km. This distance is then converted 

103 into a number of grid squares, which does not have to be an 

104 integer. Assumes the grid spacing is the same in the x and y 

105 directions and raises an error if this is not true. See 

106 SpatiallyVaryingWeightsFromMask for more details. 

107 

108 Returns: 

109 iris.cube.Cube: 

110 Merged and blended Cube. 

111 

112 Raises: 

113 RuntimeError: 

114 If calc_method is linear and cval is not None. 

115 RuntimeError: 

116 If calc_method is nonlinear and either y0val and ynval is not None. 

117 RuntimeError: 

118 If calc_method is dict and weights_dict is None. 

119 """

122 

123 if not new_name and not vicinity_radius: 

124 raise ValueError( 

125 "Neither new_name nor vicinity_radius have been specified. Use weighted-blending " 

126 "instead." 

127 ) 

128 

129 result_cube = weighted_blending.process( 

130 *cubes, 

131 coordinate=coordinate, 

132 weighting_method=weighting_method, 

133 weighting_coord=weighting_coord, 

134 weighting_config=weighting_config, 

135 attributes_config=attributes_config, 

136 cycletime=cycletime, 

137 y0val=y0val, 

138 ynval=ynval, 

139 cval=cval, 

140 model_id_attr=model_id_attr, 

141 record_run_attr=record_run_attr, 

142 spatial_weights_from_mask=spatial_weights_from_mask, 

143 fuzzy_length=fuzzy_length, 

144 ) 

145 update_name_and_vicinity_coord(result_cube, new_name, vicinity_radius) 

146 return result_cube
