31 """Training a model using Quantile Regression Random Forest. 

32 

33 Loads in arguments for training a Quantile Regression Random Forest (QRF) 

34 model which can later be applied to calibrate the forecast. 

35 Two sources of input data must be provided: historical forecasts and 

36 historical truth data (to use in calibration). The model is output as a pickle file. 

37 

38 Args: 

39 file_paths (cli.inputpaths): 

40 A list of input paths (in any order) containing: 

41 - The path to a Parquet file containing the truths to be used 

42 for calibration. The expected columns within the 

43 Parquet file are: ob_value, time, wmo_id, diagnostic, latitude, 

44 longitude and altitude. 

45 - The path to a Parquet file containing the forecasts to be used 

46 for calibration. The expected columns within the Parquet file are: 

47 forecast, blend_time, forecast_period, forecast_reference_time, time, 

48 wmo_id, percentile, diagnostic, latitude, longitude, period, height, 

49 cf_name, units. Please note that the presence of a forecast_period 

50 column is used to separate the forecast parquet file from the truth 

51 parquet file. 

52 - Optionally, paths to NetCDF files containing additional preictors. 

53 feature_config (dict): 

54 Feature configuration defining the features to be used for quantile 

55 regression. The configuration is a dictionary of strings, where the 

56 keys are the names of the input cube(s) supplied, and the values are a list. 

57 This list can contain both computed features, such as the mean or standard 

58 deviation (std), or static features, such as the altitude. The computed 

59 features will be computed using the cube defined in the dictionary key. 

60 If the key is the feature itself e.g. a distance to water cube, then the 

61 value should state "static". This will ensure the cube's data is used as 

62 the feature. The config will have the structure: 

63 "DYNAMIC_VARIABLE_CF_NAME": ["FEATURE1", "FEATURE2"] e.g: 

64 { 

65 "air_temperature": ["mean", "std", "altitude"], 

66 "visibility_at_screen_level": ["mean", "std"] 

67 "distance_to_water": ["static"], 

68 } 

69 parquet_diagnostic_names (list of str): 

70 A list containing the diagnostic names that will be used for filtering 

71 the forecast and truth DataFrames read in from the parquet files. The 

72 target diagnostic name is expected to be the first item in the list. 

73 These names could be different from the CF name e.g. 

74 'temperature_at_screen_level'. This is expected to be the same length 

75 as the cf_names and experiments lists. 

76 cf_names (list of str): 

77 A list containing the CF names of the diagnostics. The CF names should 

78 match the order of the parquet_diagnostic_names. The target diagnostic to be 

79 calibrated is expected to be the first item in the list. These names 

80 could be different from the diagnostic name used to identify in the 

81 parquet files. For example, the diagnostic name could be 

82 'temperature_at_screen_level' and the corresponding CF name could be 

83 'air_temperature'. This is expected to be the same length as the 

84 parquet_diagnostic_names and experiments lists. 

85 forecast_periods (str): 

86 Range of forecast periods to be calibrated in hours in the form: 

87 "start:end:interval" e.g. "6:18:6" or a single forecast period e.g. "6". 

88 The end value is exclusive, so "6:18:6" will calibrate the 6 and 12 hours. 

89 cycletime (str): 

90 Cycletime of a format similar to 20170109T0000Z used to filter the 

91 correct blendtimes from the dataframe on load. 

92 training_length (int): 

93 The length of the training period in days. 

94 experiments (list of str): 

95 The names of the experiment (step) that calibration is 

96 applied to. This is used to filter the forecast DataFrame on load. 

97 This is expected to be the same length as the parquet_diagnostic_names 

98 and cf_names lists. 

99 n_estimators (int): 

100 Number of trees in the forest. 

101 max_depth (int): 

102 Maximum depth of the tree. 

103 max_samples (int | float): 

104 If an int, then it is the number of samples to draw from the total number 

105 of samples available to train each tree. Note that a 'sample' refers to 

106 each row within the DataFrames constructed where each row will differ 

107 primarily based on the site, forecast period, forecast reference time and 

108 realization or percentile. If a float, then it is the fraction of samples 

109 to draw from the total number of samples available to train each tree. 

110 If None, then each tree contains the same number of samples as the total 

111 available. The trees will therefore only differ due to the use of 

112 bootstrapping (i.e. sampling with replacement) when creating each tree. 

113 max_features (int | float): 

114 If a float, then it is the fraction of features to consider when looking 

115 for the best split. If int, then it is the number of features that will 

116 be considered at each split. If None, then all features are considered. 

117 random_state (int): 

118 Random seed for reproducibility. 

119 transformation (str): 

120 Transformation to be applied to the data before fitting. 

121 pre_transform_addition (float): 

122 Value to be added before transformation. 

123 unique_site_id_keys (str): 

124 The names of the coordinates that uniquely identify each site, 

125 e.g. "wmo_id" or "latitude,longitude". 

126 kwargs: Additional keyword arguments for the quantile regression model. 

127 Returns: 

128 A quantile regression random forest model with associated transformation and 

129 pre-transformation addition that will be stored as a pickle file. 

130 """ 

131 

132 from improver.calibration.load_and_train_quantile_regression_random_forest import ( 

133 LoadForTrainQRF, 

134 PrepareAndTrainQRF, 

135 ) 

136 

137 forecast_df, truth_df, cube_inputs = LoadForTrainQRF( 

138 experiments=experiments, 

139 feature_config=feature_config, 

140 parquet_diagnostic_names=parquet_diagnostic_names, 

141 cf_names=cf_names, 

142 forecast_periods=forecast_periods, 

143 cycletime=cycletime, 

144 training_length=training_length, 

145 unique_site_id_keys=unique_site_id_keys, 

146 )(file_paths) 

147 if forecast_df is None or truth_df is None or cube_inputs is None: 

148 return None 

149 

150 kwargs = {} 

151 if max_features is not None: 

152 kwargs["max_features"] = max_features 

153 result = PrepareAndTrainQRF( 

154 feature_config=feature_config, 

155 target_cf_name=cf_names[0], 

156 n_estimators=n_estimators, 

157 max_depth=max_depth, 

158 max_samples=max_samples, 

159 random_state=random_state, 

160 transformation=transformation, 

161 pre_transform_addition=pre_transform_addition, 

162 unique_site_id_keys=unique_site_id_keys, 

163 **kwargs, 

164 )(forecast_df, truth_df, cube_inputs) 

165 if result == (None, None, None): 

166 return None 

167 

168 return result
